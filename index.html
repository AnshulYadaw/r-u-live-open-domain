How to Register Your Subdomain

  1. Star and fork our GitHub repository
  2. Create a new JSON file in the /domains folder with your desired subdomain name (example: mysite.json)
  3. Format your file according to our guidelines:
{
  "description": "Your project description",
  "domain": "r-u.live",
  "subdomain": "your-subdomain",
  
  "owner": {
    "repo": "https://github.com/yourusername/your-repo",
    "email": "your.email@example.com"
  },
  
  "record": {
    "A": ["192.0.2.1"],
    "CNAME": "your-site.com"
  },
  
  "proxied": true
}

  2. Wait for approval (usually within 24 hours)
  3. Once merged, your subdomain will be active!

Record Types Explained

Depending on your needs, you can use different types of DNS records:

  • A Record: Points your subdomain to an IPv4 address (e.g., server IP)
  • CNAME Record: Points your subdomain to another domain (e.g., GitHub Pages)
  • MX Record: For email services
  • TXT Record: For verification and SPF records

Not sure what to use? Ask in our discussions

Frequently Asked Questions

How long does it take to activate my subdomain?

After your pull request is merged, it usually takes 5-15 minutes for the DNS changes to propagate, but it can take up to 24 hours in some cases.

Can I update my subdomain later?

Yes! Simply submit a new pull request with your updated JSON file.

Is SSL/TLS supported?

Yes, all subdomains automatically get SSL/TLS support through Cloudflare when proxied is set to true.

Are there any usage limits?

We don't impose strict limits, but excessive bandwidth usage may be reviewed. This service is intended for personal and small project use.
